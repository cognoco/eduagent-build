In progress — full transparency

Latest gets the solid blue ring + "Latest" chip → that's what Continue learning resumes. Older started-but-not-finished topics appear below in a softer style with Paused chips + time-since-last-touched. They're still tappable.

Why I like this rule:
  • Honest — if I started it and didn't finish, it IS in progress
  • One simple rule — no fuzzy heuristics, easy to implement and easy to explain
  • Gentle nudge — if I have 4 paused topics, the scroll weight makes me notice I'm leaving threads hanging. Not shaming, just visible.
  • CTA stays predictable — "Continue" always resumes the Latest one. The Paused ones are tap-to-resume.
Edge cases this handles cleanly:
  • Fresh book → no In progress section. Up Next becomes the primary CTA target.
  • One in-progress → exactly the previous mockup (single active row).
  • Many in-progress → list grows; if more than ~4, we could add a "see all paused (N)" collapse. But 4+ is already a signal.
  • Just finished a topic → it moves out of In progress (since completed), CTA auto-retargets to Up Next.
