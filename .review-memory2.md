# Memory Phase 2 Review Resolution

Date: 2026-05-06
Branch: memory2

The memory architecture Phase 2 implementation was reviewed during this work
session. The actionable findings were addressed before the final code batches:

- Restored the root mobile dependencies that had been removed from
  `package.json` / `pnpm-lock.yaml`.
- Replaced the hollow pgvector null test with real nullable helper coverage.
- Changed relevance retrieval so small profiles with fewer than `k` embedded
  facts still use relevance ranking instead of falling back to recency.
- Added typed embedding failure classes for Voyage failures.
- Added idempotent `embedding IS NULL` protection to the on-write embedding
  update path.
- Added structured per-row failure logging to the memory-facts embedding
  backfill cron.
- Reused the shared pgvector formatter for batch vector updates.
- Added cross-profile integration coverage for relevance retrieval scoping.
- Updated the spec to match the implemented relevance defaults and fallback
  behavior.

Known follow-up not completed in this batch:

- Add an attempts counter or failure table for poison-pill embedding rows so
