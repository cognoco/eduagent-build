---
name: Autonomous speccing
description: When batch-speccing multiple items, make design decisions autonomously — only involve user for genuinely big trade-offs
type: feedback
---

When working through a batch of spec items, decide the best solution yourself. Only involve the user when there are genuinely large clarification or implementation differences between solutions.

**Why:** User doesn't want to be a bottleneck on 12+ specs. They trust the agent's judgment on straightforward design decisions.

**How to apply:** During brainstorming/speccing sessions with multiple items, compress the clarification phase. Present the design directly with your recommendation baked in. Only ask when the trade-off is genuinely unclear or high-stakes.
