# Identity Foundation — A-vs-B Audit, Track 2: Workstream × Layer Effort Estimates

**Date:** 2026-06-05
**Audit:** `identity-foundation/audit-a-vs-b` — Track 2 (cross-workstream synthesis)

This file synthesizes Track 1's cell grid into per-(workstream × layer) effort estimates comparing **Option A** (ship the full three-category identity model — Cat-1 adults, Cat-2 consent-capable minors, Cat-3 below-digital-consent / COPPA-under-13 children, with verifiable-parental-consent) against **Option B** (defer Cat-3 to a v2 reintroduce, ship Cat-1 + Cat-2 only). For every cell it records A-effort, B-effort, the delta direction, *where* the delta physically lives in the codebase/canon, what each option preserves, what B defers (with the v2 reintroduce cost, which is **never** folded into the v1 delta), a confidence rating, the falsifier that would move the cell, and Track-1 anchors. The headline the PM should carry away: **B's savings are real but bounded and concentrated** — the identity *spine* (consent_grant, consent_receipt, `person.birth_date`, the daily age-crossing sweep, org/`is_owner` re-anchor) is consent-category-agnostic and built under both options, so B trims COPPA/under-13/VPC surfaces around the edges rather than carving out a whole tier.

Effort taxonomy: `XS < S < M < L < XL`. Delta taxonomy: `B << A` (large saving), `B < A` (partial saving), `B = A` (no saving), `n/a` (no work in either option). Confidence: `high | medium | low`.
