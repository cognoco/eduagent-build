A program this size feels unmanageable when you watch it at the wrong altitude. This page exists so two people can run eleven initiatives without holding any of it in their heads.

01The shape: four altitudes, watch one

All work in the program lives in a four-level hierarchy. The trick: each level is somebody else's job to track, except one.

ProgramThe umbrella1 of these. The roster + queue. Jorn steers.
InitiativePRG-NN rows11 of these. A coherent body of work with one outcome and one gate.
BundleClusters of work~50 of these. Thematic chunks inside an initiative — “the string sweep”, “the auth fixes”. This is your altitude.
Work itemCosmo WIsDozens. Agents claim, execute, and close them. You never need the list.

Bundles are stable — a work item churns daily, a bundle changes only when real scope changes. If you can say "the modal-focus bundle is done, auth fixes are waiting on W2", you have full situational awareness.

02Nobody picks from a hat: gates do the scheduling

The program has no calendar and no priority meetings. Sequencing is done by five named events — the gate rail on the Sequence page. Each waiting initiative is armed behind exactly one of them. When a gate fires, it unlocks specific initiatives; if two unlock at once, the only human call is "which first, given our attention this week".

This means the answer to "what should we start next?" is never an opinion — it's printed on the rail.

Only three things ever need a human decision: ① when a gate fires — start the unlocked initiative now or briefly defer it (attention budget); ② product rulings when asked (like the WI-587 sheet, or the topic-locking question); ③ brand-new work that fits no initiative — it lands on the roster's intake line for the next touch.

03Activating an initiative is a recipe, not a project

Every waiting initiative already has a ratified charter — its scope, bundles, size, and how much supervision it needs. We deliberately do not break initiatives into work items ahead of time (plans go stale; the charter is the right level until the work is real). When its gate fires, activation is ~an hour of mechanical work:

STEP 1
TrackerOne markdown file: charter, sequence, where to pick up. The durable entry point.
STEP 2
WorkstreamA container in Cosmo so the work items have a home and an order.
STEP 3
SliceCharter bundles → PR-sized work packages in Cosmo. Pure transcription.
STEP 4
ExecuteAgents claim → build → review → close. You shepherd at bundle level.

Identity Foundation went through this exact pipeline first (on purpose — first one through proves the machinery). Every later initiative reuses the pattern.
