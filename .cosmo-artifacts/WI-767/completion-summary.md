# WI-767 — Converge activity-ledger to derive-on-read + remove dead schema (MMT-ADR-0022 arch E)

## What was done:

Converged the `mentor_activity_ledger` to MMT-ADR-0022 **architecture E** (moments
derived on read; table is seen-state + genuinely-non-reconstructable capture only):

- **Pruned the 5 declared-but-unwritten LedgerKinds** (`topic_mastered`,
  `retention_due`, `needs_deepening_added`, `recap_ready`, `snapshot_ready`) from
  `@eduagent/schemas`. These kinds implied the "materialize everything" model that
  arch E rejects — the `/now` feed already derives retention on read and
  topic-mastery lives on the session/assessment records.
- **Removed the dead schema** (absorbed WI-766): dropped the `visibility` column +
  `ledger_visibility` pgEnum (self-only; enforced by profile scope) and the
  `template_key` column (copy is resolved at read time; `/now` re-derives) via a
  committed forward migration.
- **Recorded the `session_filed` decision:** it **stays materialized** as the single
  genuinely-materialized kind. It is the seen-state anchor / residual best-effort
  write surface the ADR keeps; it is not derivable from another record. Derive-on-read
  is the documented default for *new/derivable* moments (read-time projection in
  `now-feed.ts`), not a new writer.

## What changed:

- `packages/schemas/src/activity-ledger.ts` — pruned kinds from `ledgerKindSchema` +
  `ledgerKindParamsSchema`; removed `ledgerVisibilitySchema` and
  `ledgerTemplateKeySchema`; added MMT-ADR-0022 reference comment.
- `packages/database/src/schema/activity-ledger.ts` — dropped `ledgerVisibilityEnum`,

  GDPR data-export ledger-row schema (second consumer; `export.ts` parses through it).

Writers / readers:
- `apps/api/src/services/activity-ledger.ts` (`writeActivityMoment`) — dropped
  `templateKey` input + `visibility: 'self'` insert.
- `apps/api/src/inngest/functions/auto-file-session.ts` and
  `apps/api/src/services/snapshot-aggregation.ts` — dropped the `templateKey` arg.
- `apps/api/src/services/now-feed.ts` — removed `ledgerTemplateKey` param passthrough
  and the dead deep-link branch for the pruned kinds (+ orphaned locals). The NowCard's
  own derived `candidate.templateKey` (read-time display key) is unrelated and left intact.

Migration:
- `apps/api/drizzle/0127_fast_blonde_phantom.sql` (NEW) — `DROP COLUMN template_key`,
