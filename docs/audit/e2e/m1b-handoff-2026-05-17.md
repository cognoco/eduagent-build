## Step 3 — pr-blocking expansion (per-candidate loop)

For each candidate row in `m1b-pr-blocking-candidates.md`:

1. Run the flow twice in a row on a clean Pixel API 34 emulator.
2. Time the elapsed seconds. Reject if > 90s (keep at `smoke`).
3. If both runs pass: add `- pr-blocking` to the flow's `tags:` block.
4. Run `bash scripts/validate-maestro-flows.sh` — must still exit 0 (the tag is already in the registry, so this is just a guardrail).
5. Append a row to the "Verification log" table at the bottom of `m1b-pr-blocking-candidates.md` with date, runner, both run outcomes, elapsed, and decision.
6. Stop when `pr-blocking` reaches ~20 flows or the combined budget exceeds 8 min.
