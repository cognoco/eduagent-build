# Per-turn Answer Evaluation Signal — Implementation Plan

**Goal:** Add a feature-gated, structured per-turn answer evaluation that is preserved across every session exchange path, persisted compatibly, and used for bounded rung de-escalation.
**Approach:** Extend the shared LLM envelope first, then thread one default-closed route-resolved flag through prompt construction, parsing, fallbacks, and persistence. Make canonical answer evaluations authoritative over legacy booleans, derive rung-local streaks from the latest contiguous rung visit, and express rung movement as an explicit up/hold/down decision so downscaffolding is observable and cannot be mistaken for escalation.

## Tasks

- [x] T1: Define the shared answer-evaluation contract — add optional `signals.answer_evaluation` with correctness exactly `correct | partial | incorrect | na` and optional concept length 1–200; done when the named schema tests first fail and then pass for every enum member, malformed correctness, missing/empty/oversize concept, and omission.
- [x] T2: Add the default-closed runtime flag and route binding — declare `ANSWER_EVALUATION_RUNTIME_ENABLED` as `z.enum(['true', 'false']).default('false')`, expose a helper whose only enabled input is the string `true`, type the Worker/session bindings, resolve it independently at `/messages` and `/stream`, and pass the boolean into both service entry points without changing deployed config; done when config and route-facing service tests demonstrate false-by-default, explicit true, explicit false, and both endpoint paths.
- [x] T3: Gate ordinary-loop prompt instructions — when enabled outside recitation, app-help, and active Challenge Round, require classification of the current learner message against only the immediately preceding ordinary learning question; require `na` on a first turn or any non-gradable response, while active Challenge Round continues using `challenge_round_evaluation`; done when `exchange-prompts.test.ts` first fails and then passes for enabled ordinary correct/partial/incorrect/na guidance plus disabled, first-turn, app-help, recitation, and active-Challenge exclusions.
- [x] T4: Parse, guard, and surface the canonical signal — extend parsed exchange results with `answerEvaluation`, remove it in app-help/recitation/active-Challenge guards, and leave it absent after malformed envelopes or when the flag is disabled; done when `exchanges.test.ts` and `app-help-map.test.ts` first fail and then pass for all classifications, parser failure, disabled mode, and exclusions.
- [x] T5: Preserve and persist the signal on every production path — thread the flag through `prepareExchangeContext`, non-streaming `processExchange`/`processMessage`, streaming `streamExchange`/completion, and both stream-to-nonstream fallbacks; persist `metadata.answerEvaluation`, surface it on typed SSE `done` frames, map compatibility `correctAnswer` to true only for correct and false for partial/incorrect, omit it for na/absent, and force `partialProgress=true` for canonical partial even if legacy output disagrees; done when `exchanges.test.ts`, `session-exchange.test.ts`, `session-stream-response.test.ts`, and `stream-fallback.test.ts` first fail and then pass for non-stream, stream, mid-stream fallback, pre-stream fallback, conflicts, exclusions, disabled mode, and legacy rows.
- [x] T6: Make correctness streaks canonical and visit-local — compute a capped trailing streak from canonical metadata first and legacy `correctAnswer` second, reset on partial/incorrect, skip na/unevaluated turns, and stop at a rung boundary within the latest contiguous rung visit; derive rung question and partial-hold counts from that same visit so earlier visits cannot bounce the learner; done when `session-exchange.test.ts` first fails and then passes for mixed legacy/canonical sequences, cap five, skips, resets, rung transitions, and repeated visits to the same rung.
- [x] T7: Apply explicit bounded downscaffolding — extend escalation decisions with an explicit `up | hold | down` action/direction; with the feature enabled and four completed correct answers in the current visit, lower exactly one rung with a floor at rung 1, while app-help/recitation/active Challenge freeze first, an explicit current stuck response takes precedence upward, and all existing upward behavior remains unchanged; done when `escalation.test.ts` and `session-exchange.test.ts` first fail and then pass for precedence, threshold, floor, disabled behavior, visit isolation, and unchanged upward cases.
- [x] T8: Persist and emit truthful movement audit — for down decisions, record from-rung, to-rung, streak, direction/action, and reason; use de-escalation wording/telemetry rather than escalation/offered wording, suppress offer prompt/telemetry on every actual movement, forward the decision fields through streaming and non-streaming persistence, and retain existing upward audit behavior; done when session service tests first fail and then pass for down and up persistence/logging on both paths.
- [x] T9: Prove the real persistence boundary and eval contract — add a dedicated real-database integration covering canonical metadata and compatibility mapping for correct, partial, incorrect, and na across non-stream and stream persistence, and add eval cases for enabled correct/partial/incorrect/na plus disabled omission; done when the dedicated integration test passes and Tier-1 snapshots explain only intended prompt changes in addition to the pre-existing homework-notice baseline drift captured before implementation.
