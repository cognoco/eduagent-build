System as of 02.07: 
The 4 Strands model, as actually built

  What it is

  Not a separate feature — an alternate pedagogyMode ('socratic' | 'four_strands') on the same chat/session engine. A language subject swaps the Socratic ladder prompt for buildFourStrandsPrompt() and loads the learner's mastered vocabulary. Everything else — session lifecycle, envelope, assessment engine — is shared Socratic machinery with language-specific prompts bolted on.

  How it's triggered

  Three entry paths to four_strands (session-exchange.ts:2273):

  1. Subject creation (the sound path). createSubject → detectLanguageSubject (language-detect.ts:39): a deterministic language-name hint gate then an LLM double-check whose prompt explicitly rejects "history/culture topics like French Revolution." Conservative and correct. Sets subjects.pedagogyMode='four_strands' + languageCode. This is the only place the mode is persisted.
  2. Bound session (trustworthy). Reads the subject's stored pedagogyMode.
  3. Freeform first-message regex fast-path (the leaky one). LANGUAGE_REGEX (session-exchange.ts:168) — a pure text pattern, no semantic check — flips exchange 0 of a freeform chat into four_strands. It misfires: "what happened in French history" or "what's the boiling point in Celsius" both match. Damage is limited (vocab query no-ops when no subject is resolved yet), but the first turn's behavior runs in language mode even when the real subject later classifies as History. → Hole #3.

  How the LLM behaves

  buildFourStrandsPrompt (language-prompts.ts:30-85) — a direct teacher, explicitly not Socratic:
  - Direct correction: show the corrected form, explain briefly, ask for a retry. Explicitly bans "Not yet" / Socratic withholding.
  - Vocabulary tracking: make new words explicit, recycle before adding, target 95-98% known language for input (the only hard number in the whole prompt).
  - Voice/fluency: encourage speaking; run timed drills via ui_hints.fluency_drill in the envelope.

  Your specific questions, answered

  Portion of vocab / grammar / speaking / listening?
  There is no defined portion. The prompt says only "Balance meaning-focused input, meaning-focused output, language-focused learning, and fluency development" — qualitative, no quotas, no turn targets, no self-audit. The effective portion is whatever the model improvises, and the only strand with real machinery behind it is vocabulary. → Hole #1 (biggest).

  How do we encourage speaking? One prompt line + timed fluency drills. Voice is genuinely wired: STT via expo-speech-recognition (use-speech-recognition.ts) and TTS via expo-speech, with the target locale passed through (session/index.tsx:1576). But it's a generic mic-in/TTS-out transport shared with journal/notes/homework — there's no pronunciation scoring, no accent grading, no output-quality measurement. Speaking is a channel, not an evaluated strand.

  Do we repeat spoken words / use recitation for this? No. There's a general-topic "recitation/recall-test" activity, but it's not wired to language. No repeat-after-me, no shadowing, no pronunciation drill exists anywhere. → Hole #4.

  How does the model evaluate and explain?
  - In-turn: direct correction (above), and it self-scores fluency drills (correct/total) inline in the envelope — no server-side grading; the LLM's own number is trusted.
  - At session close: reuses the Socratic assessment pipeline with a language-specific eval prompt appended (assessments.ts:132). Same masteryScore >= 0.6 pass logic, no language-specific mastery model. Post-session it does extract vocabulary (LLM, capped 8/session → vocabulary table).
  - Progress: CEFR level is live-derived (not a stored field, not dead) from mastered vocabulary counts per milestone (language-curriculum.ts:465). Drill scores are persisted (session_events + practice_activity_events — migration 0062).

  Ranked holes

  1. No strand balance is defined or measured. "Balance" is a vibe. Vocabulary dominates because it's the only strand with a backend; input/output/fluency are prompt-deep. A learner could do 20 sessions that are 90% vocab drilling and nothing enforces or even reports otherwise.
  2. Input strand is prompt-only. No graded reader / listening-passage generator tied to known vocabulary; the "95-98% comprehensible input" rule is unenforced. This is the strand most core to Nation's model and it has zero machinery.
  3. Regex trigger misfires on non-language freeform first messages (History/science phrasing). Behavioral only, but it means the model can open in language-teacher mode on the wrong subject.
  4. No speaking evaluation and no recitation. Voice is a transport with no pronunciation/output-quality feedback loop, and repeat-after-me / shadowing (the most requested beginner speaking scaffold) doesn't exist.
  5. Real prompt bug — Socratic instructions can leak into a language session. The EVALUATE ("Devil's Advocate") and TEACH_BACK ("Feynman — never correct the learner") blocks (exchange-prompts.ts:1179-1231) are not gated by isLanguageMode, and session-exchange.ts:2106 can auto-select those verification types for any learning session regardless of pedagogy. Result: a language turn can receive both "correct directly, immediately" and "never correct the learner" in the same prompt. Untested — no eval snapshot hits it.
  6. Evaluation is Socratic-shaped. Language mastery runs through the same masteryScore >= 0.6 topic-assessment engine; there's no CEFR-native or four-strands-native competency model.
