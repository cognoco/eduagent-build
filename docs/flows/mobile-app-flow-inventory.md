## Status legend

Each row is prod-active unless tagged. Tags (shared with `learning-path-flows.md`):

- **prod-active** — running for real users today.
- **flag-gated** — code shipped, a flag keeps it dark/limited in at least one environment (e.g. `MODE_NAV_V1_ENABLED`, `CHALLENGE_ROUND_RUNTIME_ENABLED`).
- **code-only** — flow present in app code, no dedicated E2E flow file yet (coverage tag, not a liveness tag).
- **orphaned** — screen/engine exists and works, but no in-app navigation reaches it (deep-link only).
- **server-built/mobile-dormant** — API path complete end-to-end, no mobile surface reaches it.
- **prompt-only** — LLM prompt rule with no UI/route surface.
- **data-only** — backend computes/persists, no UI consumes it yet.

Audience codes in the Audience column: **solo** = solo-owner learner · **child** = child on parent's account (non-proxy) · **sup** = supporter/guardian owner · **proxy** = parent-proxy ("viewing as child").
