## Local-API blocker

**Command:**
```bash
doppler run --project mentomate --config stg -- pnpm run test:e2e:web
```

**Mode:** Default Playwright config — starts `pnpm --dir ../api exec wrangler dev --port 8787` as a webServer, expects API traffic at `127.0.0.1:8787`. With `apps/api/.dev.vars` present (via `pnpm env:sync` earlier today), wrangler dev no longer errors on missing `GEMINI_API_KEY`.

**Outcome:** 1 failed (setup) / 25 did not run / 5 passed (smoke-auth only — no setup dependency).

**Root cause:** The Playwright webServer starts wrangler dev correctly, but **the prebuilt web bundle has `EXPO_PUBLIC_API_URL` baked in at build time** — pointing at staging (`mentomate-api-stg.zwizzly.workers.dev`), not localhost. Verified by:
- `grep -E "GET|POST" log` against wrangler dev shows zero non-`/v1/health` traffic
- The post-sign-in error context shows the page reached `/home` (so auth worked against staging) but `library` failed to load — symptomatic of bundle/API state hybrid

Wrangler dev is started but nothing talks to it. The "local API" Playwright config option is a misnomer in its current form: it spins up a dev API that the bundle never calls.

**To make local-API actually work would require:**
1. Set `EXPO_PUBLIC_API_URL=http://127.0.0.1:8787` *before* the web bundle is built
2. Re-run the export step (`expo export` or whatever feeds `serve-exported-web.mjs`)
3. Then start the test run

None of this is documented. CLAUDE.md just says `doppler run -c stg -- pnpm run test:e2e:web` — which silently exercises the broken path.

**Fix candidates (out of scope for the baseline; for the proposal):**
- Document and script a "true local" mode that rebuilds the bundle with localhost
- Or remove the `shouldStartLocalApi` branch entirely — the suite has effectively been remote-staging-only since the bundle started being prebuilt
- Either way: `apps/mobile/e2e-web/README.md` (P1 deliverable in scope-proposal-2) needs to call this out

## Notes on infra friction encountered

For the P4 "first-time fresh-checkout" walkthrough deliverable, the sequence that actually worked today was:

1. `doppler login` (one-time, machine-scoped)
2. `doppler setup` → pick `mentomate` / `stg` (one-time, repo-scoped)
3. `pnpm install --frozen-lockfile` (pulled a missing `@tanstack/react-query-persist-client` — lockfile-vs-tree drift)
4. `pnpm exec playwright install chromium` (browsers were on a stale version; current install pulled 1194)
5. `pnpm env:sync` — generated `.env.development.local`, `apps/api/.dev.vars`, `apps/mobile/.env.local`
6. The `pnpm run test:e2e:web` command from CLAUDE.md as-is silently runs the broken local-API path. Either pass the `PLAYWRIGHT_SKIP_LOCAL_API=1` + `PLAYWRIGHT_API_URL=...` overrides explicitly (as this baseline did), or accept that local-API needs a bundle rebuild first.

Total time to get from fresh state to first green smoke spec: roughly 5 minutes once doppler was set up. Total time to a complete full-suite run including setup: ~35 minutes (5 setup + 23 test + ~7 of debug-and-retry iterations counted in this session). The current state is consistent with "1 hour to first green smoke" being achievable; the local-API blocker is the longest remaining yak.
