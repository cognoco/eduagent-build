# Slice 1.5 PR 1.5b — Mentor-Memory Entry-Point Cue On Session Summary

**Date:** 2026-05-07
**Status:** Draft plan, ready to implement
**Branch:** TBD
**Parent plan:** 2026-05-07 conversation. Mitigation #3 for "feeling less seen" risk introduced when the per-subject interview screen goes away — make the mentor-memory surface discoverable so users who care about feeling seen can find what the tutor knows about them.
**Wave:** Slice 1.5 Wave 2 — independent of 1.5a, but value is higher once 1.5a's signal-extraction is feeding mentor-memory (i.e., once 1.5c lands).
**Size:** S

---

## Goal

Add a discoverable "Here's what your tutor knows about you" entry-point on the **session summary** screen so the mentor-memory surface is visible to users without forcing it. Quiet, optional, tappable. No push, no notification, no tooltip.

This closes the "first 30 seconds feeling less seen" risk identified in the 2026-05-07 conversation: today's interview screen visibly signals "the system is paying attention to you." Once the interview screen is gone (Slice 1.5c), users have less perceptible evidence that the system is gathering signal. A small, discoverable cue on session-summary repairs this without re-introducing intake friction.

## Current state (verified 2026-05-07)

### Session summary screen

`apps/mobile/src/app/(app)/session-summary/[sessionId].tsx` already exists. It renders, among other things, a `session-next-topic-card` (LEARN-07 in the inventory, lines 1034-1074 of the screen file). The recap layout is the natural home for a "Here's what we know about you" tile — it sits in the same "what just happened / what's next" frame.

### Mentor-memory routes

- Self: `/(app)/mentor-memory` (`mentor-memory.tsx`)
- Parent-on-child: `/(app)/child/[profileId]/mentor-memory` (`child/[profileId]/mentor-memory.tsx`)

Both are reachable today only from More tab → Mentor Memory section (ACCOUNT-15, ACCOUNT-16 in the inventory).

### What's missing

- No surface on session-summary or anywhere else proactively discovers mentor-memory.
- No place where the user sees a summarised view of "what the tutor knows" — `mentor-memory.tsx` is the source of truth, but reaching it requires intent.

## Implementation steps

1. **Build `MentorMemoryCue` component.** Single Pressable with title + subtitle + Ionicons chevron. testID `session-summary-mentor-memory-cue`. Minimum touch target 44×44 (consistent with the existing back button at line 142 of accommodations.tsx etc.).
2. **Slot it into session-summary.** Render after the recap content and `session-next-topic-card`. Tap → `goBackOrReplace` is the wrong primitive (not back navigation); use `router.push` to mentor-memory route. Determine self vs. parent-proxy from the active profile lens (existing pattern: `useActiveProfileRole`).
3. **Quiet rendering rule.** Hide the cue when:
   - The user has fewer than 2 completed sessions on this profile (no "memory" worth surfacing yet — saves the cue for when the tutor actually has signal). Use the existing snapshot / progress query if available; otherwise add a tiny field to the session-summary payload.
   - Parent-proxy mode without consent: hide entirely (consent gating per existing pattern).
4. **Copy.** Title: "What your tutor knows about you." Subtitle: "Tap to review or change." Norwegian: "Hva veilederen din vet om deg." / "Trykk for å se eller endre."
5. **Tests.** Render-and-tap, parent-proxy routing, quiet-rendering rule (hide on 0/1 sessions).

---

## Out of scope

- Inline rendering of mentor-memory contents on the session-summary screen. The cue is a link, not an embedded view. Users tap to go look.
- Modifications to mentor-memory.tsx itself. Already covered by 5j.
- Push notifications or badges. The cue is discoverable, not intrusive.

## Failure Modes

| State | Trigger | User sees | Recovery |
|---|---|---|---|
| Profile has 0 prior sessions | Edge case after profile reset | Cue hidden; session-summary renders without it | Working as intended |
| Parent-proxy without consent | Consent revoked | Cue hidden | Working as intended |
| Tap routes to a stale mentor-memory state | Cache lag | User sees old data briefly | `useLearnerProfile` refetches on focus |
| Cue copy not yet translated | i18n drift | English fallback | Add key in next i18n sweep |
